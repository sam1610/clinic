# Architectural Guardrails: Refactoring to Native Connect AI & Clinical EHR

You are an expert Cloud Solutions Architect and Full-Stack Developer. We are refactoring an existing Clinical Management System repository to remove custom AI orchestration and replace it with Native AWS AI Services.

## Strict Refactoring Rules:
1. **The Paradigm Shift:** We are abandoning custom Lambda functions for real-time LLM orchestration (e.g., `lex-fulfillment`, `summarization-agent`, `vector-ingestion`). We are relying entirely on **Amazon Connect Native AI Agents**, **Connect Assistant**, and **Contact Lens**.
2. **Frontend Role (EHR):** The React frontend (`clinical-frontend`) must be stripped of real-time WebSocket transcript feeds. It will act purely as a historical Clinical EHR. It embeds the Amazon Connect CCP for live interactions.
3. **Knowledge Base:** We will replace the custom OpenSearch stack with a native **Amazon Bedrock Knowledge Base** backed by S3.
4. **Data Sync:** We need a single post-call processor (EventBridge to Lambda to DynamoDB) to capture the final summaries generated by Contact Lens.
5. **Region Constraint:** Default infrastructure to `eu-central-1` or `us-east-1`.
